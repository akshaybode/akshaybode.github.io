🏗 System Design — Data Engineering

Senior / Lead Round
How to answer: Use a 6-step framework — (1) Clarify requirements (functional + non-functional, volume, latency, SLA) → (2) High-level architecture (ingest → store → process → serve) → (3) Storage choice (S3, Snowflake, OLTP, KV) → (4) Processing (batch / stream / micro-batch) → (5) Quality, monitoring, security, cost → (6) Trade-offs & bottlenecks.

Always Discuss

  • Functional + non-functional requirements
  • Volume, velocity, variety
  • Latency SLA (real-time vs batch)
  • Schema and data contracts
  • Idempotency and exactly-once
  • Backfill and replay strategy
  • Data quality & observability
  • Security (PII, encryption, RBAC)
  • Cost & right-sizing
  • DR & disaster recovery

Common Building Blocks

  • Source: RDBMS, API, files, Kafka, IoT
  • CDC: Debezium / AWS DMS
  • Queue: Kafka / Kinesis / SQS
  • Lake: S3 (raw / curated / refined)
  • Process: Spark / Glue / EMR / Flink
  • Warehouse: Snowflake / Redshift
  • Transform: dbt
  • Orchestrate: Airflow / Step Functions
  • Serve: BI, API, ML feature store
  • Monitor: CloudWatch / Datadog / Grafana
