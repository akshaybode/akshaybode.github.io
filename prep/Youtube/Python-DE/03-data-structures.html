🧮 Data Structures

1. Lists — your bread and butter

A list behaves like a 1D array and can hold anything — numbers, strings, even other lists.

my_list = [1, 2, "Lamba", [0, "aa", "bb"]]
print(my_list[0])      # 1
print(my_list[3][1])   # "aa"  (nested indexing)
print(my_list[0:3])    # slicing -> [1, 2, 'Lamba']

Negative indexing & step

my_list = [1, 2, 3, 4, 5, 6]
print(my_list[-3:])    # last 3 -> [4, 5, 6]
print(my_list[::2])    # every 2nd -> [1, 3, 5]
print(my_list[::-1])   # reverse -> [6, 5, 4, 3, 2, 1]
What does -3 mean? Python treats it as len(my_list) - 3. Great for "fetch the last 10 digits of a phone number" type tasks.

List operations

Lists are mutable. You don't write my_list = my_list.append(x) — just my_list.append(x) changes it in place.

2. List Comprehension

Collapse a "create list → loop → append" pattern into one line. The structure is: [ output for item in iterable if condition ].

# Traditional way
new_list = []
for i in my_list:
    new_list.append(i * i)

# List comprehension (one line!)
new_list = [i * i for i in my_list]

# With conditions (only even, and not 6)
new_list = [i * i for i in my_list if (i % 2 == 0) if (i != 6)]
  • The output goes first, then the loop, then any conditions.
  • Conditions can only be applied after the iterable.
  • Both styles are valid — comprehension shines when it keeps complex code compact.

3. Dictionaries — key/value pairs

Equally important as lists. A dictionary adds context (keys) to your values, and it maps perfectly onto JSON — exactly what you get back from API calls.

my_dict = {"x": 1, "y": 2, "z": 3}
my_dict["x"] = 10        # mutable: change a value by key
my_dict.pop("z")         # remove by KEY (not by position)

print(my_dict.keys())    # dict_keys(['x', 'y'])
print(my_dict.values())  # dict_values([10, 2])
print(my_dict.items())   # key-value pairs together

Nested dictionaries

my_dict = {"demo": {"a": 1, "b": 2, "c": 3}}
print(my_dict["demo"]["b"])   # 2  -> drill in with keys
DE relevance: API responses come back as JSON = nested dictionaries. Use .items() to read key/value pairs, and chained keys to drill into hierarchy.

4. Sets — the maths area of Python

Sets store unordered data with no keys and no duplicates (auto-removed). They support mathematical operations.

a = {1, 2, 3, 4, 5}
b = {10, 11, 2, 5}

print(a.union(b))         # all unique elements from both
print(a.intersection(b))  # only common: {2, 5}
a.remove(2)               # remove an element
a.add(20)                 # add an element

s = {5, 5, 5, 5}
print(s)                  # {5}  -> duplicates removed automatically
Gotcha: f = {} creates an empty dictionary, NOT a set. To make an empty set use f = set().

5. Tuples — immutable lists

Tuples use round braces and behave like lists, but are immutable (can't be changed). Data engineers mostly work with lists, so the practical trick is to convert a tuple to a list, do your work, then convert back if needed.

my_tup = (1, 2, 3, 4, 5)

t_list = list(my_tup)     # tuple -> list
t_list.append(6)          # now you can use list methods
my_tup = tuple(t_list)    # list -> tuple (rarely needed)
Why convert? So you only have to remember list methods (append, pop) instead of memorising a separate tuple API.

✅ Key Takeaways

  • List & dictionary are the top priorities for data engineering — master them.
  • Lists: powerful slicing ([-3:], [::2], [::-1]) and are mutable.
  • List comprehension = create + loop + filter in one line.
  • Dictionaries store key/value pairs and map directly to JSON from APIs.
  • Sets auto-remove duplicates and do union/intersection; {} is a dict, use set() for an empty set.
  • Tuples are immutable — convert to a list to work with them.
