🧡 Inheritance

The most feared OOP topic — but simple once you see the flow. A child class reuses all the methods and attributes of a parent class without rewriting them.

The Idea

Like a biological family: parent → child → grandchild. If Employee needs everything Company has, don't copy-paste — just inherit it.

TypeStructure
Single levelChild inherits one parent
MultipleOne child inherits two (or more) parents
Multi-levelGrandparent → parent → child (a chain)

Single-Level Inheritance

Pass the parent class in parentheses when defining the child. Now the child can call the parent's methods too.

class Company:
    def company_info(self):
        return "Company name is XYZ"

class Employee(Company):       # inherit Company
    def employee_info(self):
        return "Employee name is Rahul"

emp1 = Employee()
print(emp1.employee_info())    # own method
print(emp1.company_info())     # inherited from Company!

The hard part: constructors

With __init__ in both classes, the child must explicitly provide the parent's attributes too — otherwise the parent's attributes don't exist on the child.

class Company:
    def __init__(self, company_name):
        self.com_name = company_name
    def company_info(self):
        print(f"Company name is {self.com_name}")

class Employee(Company):
    def __init__(self, employee_name, company_name, country):
        self.emp_name = employee_name
        Company.__init__(self, company_name)   # call parent's constructor
        self.country = country

emp1 = Employee("Rahul", "XYZ", "US")
Recommended pattern: call the parent constructor explicitly with ParentClass.__init__(self, ...). It stays consistent across single, multiple and multi-level inheritance.

super() — the alternative

super().company_info() calls the parent (one level up). It's fine for single-level, but gets confusing with multiple inheritance — so the course recommends the explicit ClassName.method(self) style instead.

# these two are equivalent in single-level inheritance
Company.company_info(self)
super().company_info()

Multiple Inheritance

One child, two parents. Inherit both by listing them, and provide both parents' attributes in the child constructor.

↘

↗
class Countryparent 2
class Employee(Company, Country):
    def __init__(self, employee_name, company_name, country_name):
        self.emp_name = employee_name
        Company.__init__(self, company_name)    # parent 1
        Country.__init__(self, country_name)    # parent 2

Multi-Level Inheritance

A chain: Company → Department → Employee. When Employee inherits Department, it automatically gets Company too. Each level passes attributes up the chain.

class Company:
    def __init__(self, company_name):
        self.com_name = company_name

class Department(Company):
    def __init__(self, department_name, company_name):
        self.department_name = department_name
        Company.__init__(self, company_name)        # pass up to Company

class Employee(Department):
    def __init__(self, emp_name, department_name, company_name):
        self.emp_name = emp_name
        Department.__init__(self, department_name, company_name)  # passes up the whole chain

emp1 = Employee("Rahul", "HR", "ABC")
emp1.all_info()        # can reach Employee, Department AND Company data
The golden rule: always pass all the attributes up the chain in each constructor — that's what makes inherited methods work.

Method Priority (Overriding)

If the same method name exists at multiple levels, Python picks the closest one first:

  • Local first: the child's own method wins ("local things get priority").
  • Then nearest parent: if the child doesn't have it, the first/nearest inherited parent's version runs.
  • Then up the chain: only if no closer class has it does the grandparent's version run.
# If Employee, Department, Company all define show_me():
emp1.show_me()   # runs Employee's version (the most local one)

✅ Key Takeaways

  • Inheritance lets a child class reuse a parent's methods & attributes — no copy-paste.
  • With constructors, the child must explicitly pass the parent's attributes (Parent.__init__(self, ...)).
  • Prefer explicit ClassName.__init__() over super() — it works the same everywhere.
  • Multiple = several parents; multi-level = a chain that auto-inherits up the line.
  • Method priority: local → nearest parent → up the chain (overriding).
