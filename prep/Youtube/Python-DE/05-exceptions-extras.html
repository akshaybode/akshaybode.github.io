🛡️ Exceptions & Power-ups

Handling errors gracefully is a non-negotiable for any pipeline. Plus a handful of small but constantly-used features: f-strings, enumerate, and global vs local variables.

Exception Handling: try / except

Python's version of try/catch. If a block of code fails, you don't want the whole program to stop — you want to log the error and carry on.

try / except flow
try:run risky code
→
error?jump to except
→
except:handle / log
→
rest of codekeeps running
try:
    if (x > 10):              # if x is a string, this fails
        print("greater")
except Exception as e:
    print("You have an error")
    print(e)                  # the actual error message

print("hello world")          # still runs!
  • except Exception as e captures the error object — print or log e to see what went wrong.
  • Without try/except the program halts at the error and the rest never runs.

finally — always runs

The third member of the group. Whatever is in finally executes no matter what — error or not. Even works after a return inside a function (the one thing that runs past a return).

def my_function(p_x):
    try:
        if (p_x % 2 == 0):
            return 1          # function exits here...
    except Exception as e:
        return e
    finally:
        print("hello world")  # ...but finally STILL runs

Raising Custom Errors

Sometimes a value is technically valid Python but invalid for your program. You can force an error with raise.

x = 99
if (x < 100):
    raise ValueError("value less than 100 is not allowed")
RaiseShows as
raise ValueError(...)ValueError
raise Exception(...)Exception
raise TypeError(...)TypeError (e.g. a typo / wrong type)
Why? Your program may have a rule like "values below 100 are forbidden". The message you write becomes the error's context — great for validation.

f-strings — dynamic strings

The cleanest way to inject variables into a string. Prefix with f and put variables in { }. Hugely useful for building dynamic cloud paths and error logs.

name = "Ansh"
my_string = f"Hello my name is {name}. How are you?"
print(my_string)     # Hello my name is Ansh. How are you?

# in error handling
print(f"Hey you got this error: {e}")

enumerate — index + value together

When looping, you often want both the position and the value. enumerate gives both as a tuple per iteration.

my_list = [100, 200, 300, 400, 500]
for i, x in enumerate(my_list):
    print(i, x)      # i = position (0,1,2...), x = value

Global vs Local Variables

A variable defined inside a function is local and takes priority there. A variable outside is global.

x = 100                # global

def f():
    print(x)           # 100 -> uses global

def g():
    x = 5              # local takes priority
    print(x)           # 5
  • A function reads the global value by default if no local variable shadows it.
  • A local variable of the same name always wins inside the function.
  • Use the global keyword to modify the global from inside (must declare it before assigning).

✅ Key Takeaways

  • try / except keeps your pipeline running when a block fails; capture details with except Exception as e.
  • finally always runs — even after a return inside a function.
  • raise your own ValueError / TypeError for program-specific validation.
  • f-strings inject variables into text — perfect for dynamic paths and logs.
  • enumerate gives index + value in one loop.
  • Local variables shadow globals inside a function.
