🎲 Conditionals & Loops

The decision-making and repetition tools you'll use everywhere. Loops are a data engineer's favourite thing — we work with big data and need to iterate over hundreds of tables.

Conditionals: if / elif / else

It's plain English: if this is true do that, otherwise do something else.

x = 10
if (x == 10):
    print("x is 10")
elif (x > 100):
    print("x is too big")
else:
    print("x is not 10")
= vs ==: A single = assigns a value. A double == compares. Use == inside conditions.
  • elif = "else if" — checked only if the previous conditions failed.
  • else = the catch-all; it always runs if nothing above matched.
  • Only one branch runs — once a condition is true, the rest are skipped.

Wrap conditions in braces

Always wrap compound conditions in ( ). It promotes readability and prevents subtle bugs.

if (x > 100) and (x < 200):
    print("between 100 and 200")
Real bug from the video: without braces, x > 100 and x < 200 created a conflict. Adding (x > 100) and (x < 200) fixed it. Add braces wherever you can.

Nested if

You can put an if inside another if to create branches within branches — common in data validation.

Loops: iteration is the data engineer's superpower

Why are loops so important? Because we work with big data — reading from many tables (orders, products, customers...). Instead of repeating code per table, we loop over them.

The for loop (you'll use it ~80% of the time)

my_list = ["orders", "products", "customers"]
for i in my_list:
    print(i)        # i takes each value in turn

range() — loop over numbers

for i in range(1, 101):   # 1 to 100  (stop is exclusive: n-1)
    print(i)

Combining loops with conditions (a real DE pattern)

You rarely process everything blindly — you validate first. Note .lower() handles inconsistent casing in real data:

for i in table_list:
    if (i.lower() == "order"):
        print("table order")    # only process the orders table
    else:
        print("no table order")

Nested loops

Because a string acts like an array, you can loop inside a loop:

for i in table_list:        # i = "order"
    for x in i:             # x = 'o','r','d','e','r'
        print(x)

break and continue

KeywordEffectDE use-case
breakImmediately exits the whole loop — doesn't even run the rest of the current iterationFound the one table you needed → stop searching
continueSkips the rest of the current iteration and moves to the nextSkip the order table, process all others
# break — stop as soon as we hit "order"
for i in table_list:
    print(i)
    if (i.lower() == "order"):
        break

# continue — skip "order", process the rest
for i in table_list:
    if (i.lower() == "order"):
        continue
    print(i)

The while Loop

Used ~20% of the time. Unlike for, a while loop does not iterate for you — you must update the counter yourself, or you get an infinite loop.

x = 1
while (x < 11):
    print(x)
    x = x + 1     # CRUCIAL: without this it loops forever
Infinite loop: forgetting x = x + 1 means the condition is always true — the program prints forever until execution is halted (out of memory).

Common pattern: while True + break

The speaker personally prefers running an "infinite" loop and breaking out on a condition:

x = 1
while True:
    print("hello")
    x = x + 1
    if (x > 10):
        break

✅ Key Takeaways

  • if/elif/else — only one branch runs; use == to compare.
  • Always wrap compound conditions in parentheses for clarity and to avoid bugs.
  • for loops are the workhorse — iterate lists, ranges, even characters of a string.
  • break exits the loop entirely; continue skips to the next iteration.
  • while loops need you to manage the counter manually — or use while True + break.
  • Combine loops with conditions to build validation into your pipelines.
