⚙️ Functions, Lambda & map / filter / reduce

Functions stop you rewriting code again and again — a non-negotiable skill. Then we meet lightweight lambda functions and the three "special forces": map, filter and reduce.

Why Functions?

Imagine reusing 50 lines of code 10 times — that's 500 lines. Register those 50 lines under a function and each reuse becomes a single call. Result: ~60 lines, plus better readability and the ability to vary behaviour via parameters.

define once, call many times
def my_func(p):50 lines once
→
my_func(20)call 1
→
my_func(30)call 2
→
my_func(40)call N

Defining & Calling Functions

def my_function(p_x):       # def keyword + name + parameter
    return p_x              # send a value back

result = my_function(20)    # call it, store the returned value
print(result)               # 20

print vs return — the crucial difference

printreturn
What it doesDisplays output on screenSends a value back to the caller
Can be stored?NoYes — in a variable for later use
In the real world you almost always return. Functions are meant to produce values you use downstream, not just print.

Default values

Give a parameter a default so the code doesn't fail if the caller omits it. A passed value always overrides the default.

def my_function(p_x, p_y=30):
    return p_x + p_y

my_function(20)      # 50  (uses default p_y=30)
my_function(20, 5)   # 25  (override)

*args and **kwargs — flexible parameters

SyntaxAcceptsBecomes a...
*argsAny number of valuestuple
**kwargsAny number of key=value pairsdictionary
def f(*p_x):          # collect many values
    print(type(p_x))  # <class 'tuple'>
f(1, 2, 3, 100)       # no limit on how many

def g(**p_x):                  # collect keyword args
    print(type(p_x))           # <class 'dict'>
    print(p_x.keys())
g(x=20, y=60, z=70)            # -> {'x':20, 'y':60, 'z':70}

Lambda Functions — lightweight one-liners

A lambda is a mini, anonymous function for simple logic. Use it when defining a full def would be overkill.

# Regular function
def addition(x, y):
    return x + y

# Equivalent lambda
addition = lambda x, y: x + y

print(addition(10, 20))   # 30
  • Syntax: lambda parameters: return_expression — no def, no indentation.
  • Best for lightweight logic; use a normal function for complex logic.

The Three Special Forces: map, filter, reduce

These apply a function across an iterable (usually a list — the most common DE iterable). The key idea: write a function for a single value, then apply it to every element.

map — apply to every element

Runs the function on each item and returns a map object — convert it to a list (or tuple).

def square(p_x):
    return p_x * p_x

my_list = [1, 2, 3, 4, 5]
result = list(map(square, my_list))   # [1, 4, 9, 16, 25]

filter — keep only what passes a condition

Like map, but only returns elements where the function is true (no Nones).

def is_even(p_x):
    if (p_x % 2 == 0):
        return p_x

result = list(filter(is_even, my_list))   # [2, 4]

reduce — collapse the list to one value

Takes the first two elements, applies the function, then keeps folding the result with the next element. Must be imported from functools.

from functools import reduce

def add(p_x, p_y):
    return p_x + p_y

result = reduce(add, [1, 2, 3, 4, 5])
# 1+2=3, 3+3=6, 6+4=10, 10+5=15  ->  15
ToolBehaviourHow often you'll use it
mapFunction on each item → new listYour go-to — master this
filterKeep items passing a conditionSometimes
reduceFold the whole list into one valueRare, but know it exists

✅ Key Takeaways

  • Functions = reusable code via def; parameters make them flexible.
  • return (storable) is more important than print (display only).
  • Default values prevent failures; *args → tuple, **kwargs → dict.
  • Lambda = one-line function for lightweight logic.
  • map is your daily driver; filter adds a condition; reduce (from functools) folds to one value.
