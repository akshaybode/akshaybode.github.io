No prerequisites, no installation. We start in an online interpreter and learn how Python actually runs your code, then master the print statement and strings — which data engineers work with constantly.

How Python Runs Your Code

People call Python an "interpreted" language, but that's only half the story. Two steps happen every time you run a .py file:

What happens when you run a .py file
Your codemain.py
→
Compile→ bytecode
→
PVMPython Virtual Machine
→
Run line by lineat runtime
  • Step 1 — Compile: the whole script is first compiled into an intermediate form called bytecode.
  • Step 2 — Interpret: the bytecode goes to the Python Virtual Machine (PVM), which executes it line by line at runtime. That line-by-line execution is why Python is called "interpreted".
Vs C / C++: those compile straight to machine code the OS runs directly, so they're much faster. That speed is why data tools like Databricks Photon and Fabric native executors are built on C++ libraries (e.g. Apache Gluten). Python wins on readability, reliability and AI-tool integration.

The print Statement (and why it teaches strings)

The classic first program. print() displays whatever you put inside it on the screen. The course uses it as a vehicle to teach strings, because data engineers work with strings constantly (building dynamic file paths, cloud URLs, etc.).

print("hello Ansh Lamba")          # one value
print("hello", 1, "I love you")    # combine values with commas
print("a", 1, "b", sep="$")        # custom separator -> a$1$b
  • Single vs double quotes: both define strings — 'hi' and "hi" are identical.
  • sep=: places a delimiter between comma-separated values.

Multi-line strings

You can't break a normal string onto a new line — Python interprets line by line and throws SyntaxError: unterminated string literal. For multi-line text use triple quotes (''' or """):

text = '''Hello, this is XYZ.
How is your day?'''
print(text)

Escaping quotes

If a quote inside the string clashes with the surrounding quotes, you have two clean options:

  • Swap the outer quotes: use double quotes outside if you need a single quote inside (best for readability).
  • Backslash escape: 'hello \'Ansh\'' — the backslash tells the interpreter "treat this as a literal character, not syntax".

Variables & Assignment

A variable is just a container (think of a jar) that holds a value. You store with a single = and access by name.

my_var = "Ansh"
x = 10
y = 5
print(x + y)              # 15  (arithmetic)
print(my_var + "Lamba")   # AnshLamba  (string concatenation)
# print(x + my_var)       # ERROR: can't add int + str

Multiple assignment

x, y, z = 10, 5, 20     # assign several at once
a = b = c = 20          # same value to all three

Comments

  • Single line: start with #.
  • Multi-line: wrap in triple quotes '''...'''.

Comments are optional but make you a good developer — imagine debugging line 20 of a 1,000-line file, or another developer reading your code months later.

Splitting a long line of code

To spread one statement over multiple lines, end each with a backslash \, or wrap the whole expression in parentheses ( ). Very handy for long Spark / data-engineering code.

total = 10 + 20 + 30 \
        + 20 + 10
# or
total = (10 + 20 + 30
         + 20 + 10)

Indentation

Unlike C++, Python requires indentation — it defines the hierarchy (parent → child). Code indented under an if belongs to that if. Wrong indentation = error.

x = 1
if x == 1:
    print("wow")     # indented = inside the if

Type Casting

Type casting = converting one data type to another. There are two kinds:

TypeWhat happensExample
ExplicitYou force the conversion yourselfint("10"), str(10)
ImplicitPython converts automatically when it's safe10 + 10.5 → int becomes 10.0 then adds → 20.5
b = "10"
print(type(b))        # <class 'str'>
b_new = int(b)        # explicit cast to int
print(type(b_new))    # <class 'int'>

print(10 + 10.5)      # 20.5  (implicit: int -> float)

Strings Behave Like Arrays: Indexing & Slicing

A string is a sequence of characters, so it behaves like an array. Counting starts at 0.

x = "Ansh"
print(x[0])     # A   (first character)
print(x[1])     # n

Slicing — [start : stop]

The stop index is excluded (goes up to n−1):

name = "Anshlamba"
print(name[0:4])   # Ansh  (indexes 0,1,2,3 -- 4 is excluded)
print(name[3:7])   # hlam
print(name[:])     # whole string (default start=0, stop=len)
print(name[0:len(name)])  # same thing
  • Omit start → defaults to 0.
  • Omit stop → defaults to n (the length, found with len()).
Remember: slicing always stops at n − 1, so to include index 6 you write [3:7]. This trips up everyone at first.

✅ Key Takeaways

  • Python compiles to bytecode, then the PVM interprets it line by line.
  • print() + commas + sep= combine values; triple quotes give multi-line text.
  • Variables are containers; you can't mix int + str without casting.
  • Indentation defines structure in Python — it's mandatory, not cosmetic.
  • Strings act like arrays — master indexing & slicing (stop is exclusive, stops at n−1).
  • Know the concept; look up the exact method name when you need it.
