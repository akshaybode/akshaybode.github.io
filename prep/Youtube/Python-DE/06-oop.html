🧩 Object-Oriented Programming

When functions and variables get scattered everywhere, your program becomes a mess. OOP bundles related functions and variables into reusable templates — the foundation for any scalable app.

Why OOP? The "mess" problem

Picture an online store with dozens of scattered functions and variables. Even with great naming you can't manage it. OOP groups related code into bundles (classes), so you can reuse and scale cleanly.

Building Your First Class

class Employee:
    emp_name = "Rahul"          # class attribute
    emp_department = "IT"

    def info(self):             # method -- needs 'self'
        return f"Employee {self.emp_name} works for {self.emp_department}"

emp1 = Employee()               # create an OBJECT
print(emp1.emp_name)            # access attribute -> Rahul
print(emp1.info())              # call method
  • Every method's first parameter must be self.
  • To use an attribute inside a method, prefix it with self. (e.g. self.emp_name).

What is self?

When you call emp1.info(), Python secretly runs Employee.info(emp1) — it passes the object itself as self. That's why each object behaves independently.

Two equivalent calls: emp1.info() is just a friendly shortcut for Employee.info(emp1).

The Constructor: __init__

Some attributes are constant (company name); others are dynamic per object (employee name). The constructor __init__ runs automatically the moment you create an object — perfect for setting up dynamic (instance) attributes.

class Employee:
    company_name = "XYZ"                       # class attribute (shared)

    def __init__(self, employee_name, employee_department):
        self.emp_name = employee_name          # instance attributes
        self.emp_department = employee_department

    def info(self):
        return f"Employee {self.emp_name} works for {self.emp_department} in {self.company_name}"

emp1 = Employee("Priya", "IT")     # __init__ runs automatically
emp2 = Employee("Priyanka", "HR")
print(emp1.info())                 # Priya / IT
print(emp2.info())                 # Priyanka / HR

Changing Class Attributes (the right way)

You can change a class attribute from outside (Employee.company_name = "ABC"), but that's not recommended — you lose track of changes. Always change it from inside the class through a dedicated method.

class Employee:
    company_name = "XYZ"
    # ...
    def changes(self, new_company):
        Employee.company_name = new_company    # change for ALL objects

emp1.changes("New Company")
Note: changing via an object (emp2.company_name = "ABC") only affects that one object. Changing via the class name affects everyone.

Static method

A "junk"/utility function that lives in the class but isn't tied to its attributes. No self.

class Employee:
    @staticmethod
    def addition(x, y):
        return x + y

print(Employee.addition(1, 2))   # 3
Common first-timer error: "takes 2 positional arguments but 3 were given" — because calling on an object auto-passes the object too. Marking it @staticmethod fixes it.

Class method

Built to interact with class attributes. Uses cls (industry standard) instead of self, so it's clear you're touching the class, not an object.

class Employee:
    company_name = "XYZ"

    @classmethod
    def change_company(cls, new_company):
        cls.company_name = new_company

Getters & Setters

Getter = returns values. Setter = sets/updates instance attributes. You can do both with normal methods, but Python has elegant decorators for them.

class Employee:
    def __init__(self, name, dept):
        self.emp_name = name
        self.emp_department = dept

    @property                         # GETTER
    def info(self):
        return f"{self.emp_name} works in {self.emp_department}"

    @info.setter                      # SETTER (same name)
    def info(self, new_details):      # one parameter (a list)
        self.emp_name = new_details[0]
        self.emp_department = new_details[1]

emp1 = Employee("Priya", "IT")
print(emp1.info)                      # GETTER -> called like an attribute
emp1.info = ["Priya", "HR"]           # SETTER -> assigned like an attribute
  • @property turns a method into a getter — call it without parentheses.
  • @info.setter defines the setter for that same name — assign to it like an attribute.
  • Setters take a single parameter, so pass multiple values as a list and unpack by index.

✅ Key Takeaways

  • A class bundles methods (functions) + attributes (variables); an object is a reusable instance.
  • Every method takes self; emp1.info() = Employee.info(emp1).
  • __init__ (the constructor) runs automatically and sets instance attributes.
  • Class attributes are shared/constant; instance attributes are per-object.
  • @staticmethod (no self) for utilities, @classmethod (cls) to touch class attributes.
  • @property / @x.setter give clean getters & setters — called like attributes.
