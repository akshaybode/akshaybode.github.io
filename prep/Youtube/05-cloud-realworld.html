1. What Is Cloud Computing?

Cloud providers are giant computers in data centers that rent you compute and services. The top three: AWS (Amazon), GCP (Google), Azure (Microsoft) (others: IBM Cloud, Oracle Cloud, Cloudera).

On-premise vs Cloud

Before cloud, you built your own servers — RAM, disks, CPU/GPU, cooling (ACs), networking, switches, routers. Problems: maintenance, hardware failure, replication, scaling, disasters (earthquake = lost data center).

Pay-per-use: the cloud's killer feature. Need a VM for 2 hours? Pay for 2 hours — no need to run hardware 24/7.

4. Which Cloud Should You Learn?

The instructor's personal opinion (may change with trends):

  • Azure — increasingly #1 for enterprise (banks, big companies already on Microsoft 365). Great DE services.
  • AWS — favorite of startups (good credits, big talent pool, easy to start — Zepto, CRED).
  • GCP — strong specific services (e.g., BigQuery); learn it if a target company requires it.
Advice for students: Pick any one of the three and you'll likely get a job. For enterprise → Azure; for startups → AWS/GCP. You can also do hybrid/cross-cloud (e.g., S3 + Databricks + BigQuery), but it's harder to maintain.

6. Real-World Case Study — Dream11

Dream11 (fantasy sports app, India) built its data platform on AWS + open source. Even though it looks complex, it maps perfectly onto the DE lifecycle.

Two pipelines

  • Batch pipeline: Kafka → S3 → Spark (ETL) → Redshift. Plus ad-hoc analysis via Amazon Athena and reporting via Looker.
  • Real-time pipeline: streaming engine Apache Flink → Elasticsearch → notifications/visualization.
The takeaway: Once you know the fundamentals, you can read any architecture in the world — spot the ingestion, transformation, storage, and serving. Tools are swappable (Redshift↔Snowflake, Glue↔Databricks); the fundamentals are constant.

8. Modern Data Stack & Reverse ETL

Modern companies say "the old tools/approach are outdated" and make life easier — connect sources, auto-load to the warehouse, transform on demand. This shifts ETL → ELT.

FunctionModern tools
IngestionFivetran, Airbyte, Stitch (or Python/SQL)
StorageBigQuery, Snowflake, Databricks
TransformationDBT (Data Build Tool) — landing → staging → warehouse → mart layers
OrchestrationApache Airflow
BILooker, Data Studio
Data qualityGreat Expectations
Metadata / lineageOpenLineage, DataHub
DBT will replace SQL? No. SQL is still the king of data — always learn it. DBT is gaining popularity for modern, layered transformations.

Reverse ETL: push transformed data back into source/operational systems to gain more insights — an emerging pattern.

How to pick any tool: Does it solve the business problem? Is it reversible (easy to swap)? Does it fit your cost/time constraints? Startups often pay for managed tools to save time; others use open-source Spark to save money.
