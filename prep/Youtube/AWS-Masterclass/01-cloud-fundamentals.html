1. What is Cloud Computing?

The word is two words glued together: cloud + computing. Break it down.

Computing

Computing = the use of a computer — a machine that can compute (process / calculate). A computer is called a computer because it bundles the parts needed to compute: RAM, ROM / storage, and a processor. Using that machine = computing. You already do this every day on your laptop.

Cloud

A cloud is just a connection / bridge between you and a machine, created over the internet. Why a bridge? Because the machine isn't in front of you — imagine your computer is sitting in a friend's house and you want to use it from your home. The internet is what connects you.

Cloud computing = computing over the internet

Definition in one line: Cloud computing is performing computing not physically — you don't own the machine in front of you; you use it over the internet.

2. What is AWS?

AWS = Amazon Web Services. Amazon has many branches (Marketplace, Prime, etc.); AWS is the branch that sells web services. AWS owns huge numbers of machines and says: "You want compute? Use my machines over the internet and pay me for it." So the "cloud" you connect to is AWS, and the machine you use is actually owned by AWS.

The Big Three (not four)

AWS was the first cloud provider, which is why it has the largest market share. The major players:

ProviderOwnerApprox. market share
AWS (Amazon Web Services)Amazon~30% (biggest)
AzureMicrosoft~20%
GCP (Google Cloud Platform)Google~10%

You could call it the "big four" if you add IBM, but realistically it's the big three. AWS leads because of early adoption, a massive catalog of services, reputation and brand. AWS also created the hype around cloud — the race only started after AWS proved it worked.

3. Why Cloud? (Buy vs Rent)

If machines are getting cheaper, why don't companies just buy their own? Picture a startup that needs several machines. They have two options:

The two options for a growing company
Option 1: Buy serversHigh cost, you manage everything
vs
Option 2: Rent from cloudPay-as-you-go, provider manages it

Problems that buying-your-own creates (and cloud solves)

  • High cost / high investment — buying many machines up front is brutal for a startup.
  • Scaling — during a festival/sale season you suddenly need far more capacity, then less afterwards.
  • Resource wastage — some months you only use 2 of your 5 machines; the rest sit idle.
  • No downtime allowed — you can't afford to wait when you need a machine immediately.
  • Data redundancy — if a data center is hit by a disaster, you need an instant backup elsewhere. Customers don't care about your data-center problems; they just want the service to work.
  • Security, firewall rules, and a hundred other concerns.
The renting analogy: Living in a rented house — you don't own it, you pay rent, and you get flexibility (change houses, discontinue, request more). In tech, renting infrastructure is the best thing: the startup focuses on its business, AWS handles the infrastructure.

4. What Cloud Actually Offers (200+ services)

Cloud started with machines/servers as its core business. But once that problem was solved, providers realized that owning a machine still means managing many things around it — for example a database needs the server downloaded, scaling handled, upgrades applied, and so on.

So they kept asking "can we solve this too?" — and because they already have the powerhouse (servers), anything that can run on a computer, they can offer as a managed service. Today AWS has 200+ services.

Relax — you don't need all 200. A data engineer might actively use ~10 services. That's why DEs must know cloud: you work with those specific services daily. These notes cover exactly the ones that matter.

5. Cloud Service Types: IaaS, PaaS, SaaS

Services are grouped by how much the cloud manages vs how much you manage. The more the cloud handles, the less control (and less work) you have.

IaaS — Infrastructure as a Service

Cloud gives you only the infrastructure (the machine, with the RAM/storage you ask for). You manage everything inside: OS, Windows upgrades, every application. Like buying an empty machine from a shop. Example: EC2.

PaaS — Platform as a Service

One step further: cloud manages the underlying infrastructure (scaling, upgrades). You just manage your platform/app — e.g. a managed PostgreSQL database where you don't touch the machine it runs on. Example: managed databases, Glue, many storage services.

SaaS — Software as a Service

The most hyped. You don't manage platform or infrastructure — you just use the final product over the internet. Examples: Amazon.com itself, Microsoft Fabric, basically any web app you log into.

Who manages what (less control → more control)

←
PaaSManage your app. AWS manages infra.

6. Cloud Models / Architectures

A separate grouping — not by management level, but by who can access the cloud.

ModelMeaningExample use
Public CloudAccessible to everyone over the internet (not restricted).AWS, Azure, GCP — the three kings.
Private CloudSame cloud concept, but locked to one specific organization. Built for sensitive data.Government / entities handling sensitive data.
Hybrid CloudA mix: build a private cloud for sensitive parts, but also use some public services for the rest.AWS EC2 for compute + private infra for secure data.
Recap of Part 1: Cloud computing = using a machine over the internet. AWS is the first and biggest cloud provider (big three: AWS, Azure, GCP). Companies rent instead of buy to avoid cost, scaling, wastage, downtime and redundancy problems. AWS has 200+ services, grouped as IaaS/PaaS/SaaS (by how much they manage) and deployed as public/private/hybrid clouds (by who can access them).
