02 — Python for Data Engineers

The golden rule: you are a data engineer, not a web developer. Learn the Python that moves and shapes data, not Django. You already have your Youtube/Python-DE notes — this is the interview-relevant distillation.

Data structures you'll actually use

# Comprehensions (transform/filter in one line)

Functions, generators, iterators (memory matters in DE)

# Generator: lazy, streams large files without loading into memory
def read_lines(path):
    with open(path) as f:
        for line in f:
            yield line.rstrip("\n")

# Process a huge file line-by-line (constant memory)
total = sum(1 for _ in read_lines("huge.csv"))
  • Know generator vs list (lazy vs eager) — classic DE memory question.
  • *args, **kwargs, default args, type hints (def f(x: int) -> str:).

Error handling & logging (production mindset)

import logging
logging.basicConfig(level=logging.INFO)
log = logging.getLogger(__name__)

try:
    result = risky_load()
except (ValueError, KeyError) as e:
    log.error("load failed: %s", e)
    raise                    # don't swallow silently in a pipeline
finally:
    cleanup()
  • Make pipelines idempotent and retry-safe; log inputs/row counts.

  • @dataclass for config/records; context managers (with) for resources.

Concurrency (know the difference)

  • I/O-bound (API/DB calls) → threading / asyncio (GIL released on I/O).
  • CPU-bound → multiprocessing (real parallelism) — but for big data, push it to Spark/the warehouse instead.

DE essentials checklist

  • requests (APIs), sqlalchemy/db drivers, pyarrow, boto3/cloud SDKs.
  • Environments: venv + requirements.txt; secrets via env vars, never hardcoded.
  • Write small, testable functions; add pytest unit tests for transforms.
