Data Engineer Interview Prep — 1-Month Sprint (Snowflake + dbt + Airflow)

Your career-switch kit, built around what you already know. Target: mid-level Data Engineer (3–6 yrs).

You are NOT starting from zero. You bring: strong SQL/Oracle/PL-SQL, Java, telecom billing/ordering domain (Amdocs — real ETL/batch/data-modeling experience), and Python in progress. This kit closes the remaining gaps and repackages your experience as data engineering.

The 4-week schedule (~3 hrs/weekday, more on weekends)

Week 1 — Base + start Spark + START APPLYING

  • Finish Python for DE (02), level up SQL for DE (01).
  • Begin PySpark fundamentals (03, first half).
  • Rewrite your resume in DE language and start applying. Do not wait until "ready."

Week 2 — PySpark deep + modeling + warehousing

  • PySpark DataFrame API + internals (03 fully).
  • Data modeling (04) + warehousing & file formats (05).
  • Daily SQL practice (1–2 hard queries).

Week 3 — Snowflake + dbt + Airflow + build THE project

  • Snowflake (06), dbt (07), Airflow (08).
  • Build the end-to-end project (project/README.md). Push to GitHub with a diagram.

Week 4 — Design + mocks + polish + apply hard

  • DE system design (09), behavioral (10).
  • Run all 4 mock rounds in mock-drills.md.
  • Polish LinkedIn, chase referrals, do final rapid revision.

If you get a 2nd month

Add a certification to signal credibility fast (Snowflake SnowPro Core or Databricks Data Engineer Associate), a streaming flavor (Kafka + Spark Structured Streaming, conceptual), and CI/CD + Docker around your pipeline.

Honest framing

You're switching fast — lead with genuine SQL/engineering strength and your telecom data background. Don't invent production history with tools you haven't touched; say how you'd approach it. Interviewers reward clear reasoning + honesty.
