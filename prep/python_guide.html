4. Object-Oriented Programming

Classes Basics

class Employee:
    company = "Acme Corp"            # class attribute (shared)

    def __init__(self, name, salary):
        self.name = name             # instance attribute
        self.salary = salary

    def annual(self):                # instance method
        return self.salary * 12

    @classmethod
    def from_dict(cls, data):        # alternative constructor
        return cls(data["name"], data["salary"])

    @staticmethod
    def is_high_earner(salary):      # no self / cls
        return salary > 100000

    def __repr__(self):              # for debugging
        return f"Employee({self.name!r}, {self.salary})"

    def __str__(self):               # for users
        return f"{self.name} earns {self.salary}"

emp = Employee("Asha", 80000)
emp.annual()
Employee.from_dict({"name": "Ravi", "salary": 90000})
Employee.is_high_earner(150000)

Inheritance

class Manager(Employee):
    def __init__(self, name, salary, team_size):
        super().__init__(name, salary)
        self.team_size = team_size

    def annual(self):                # override
        base = super().annual()
        bonus = self.team_size * 1000

Dunder (Magic) Methods

class Money:
    def __init__(self, amount, currency="INR"):
        self.amount = amount
        self.currency = currency

    def __add__(self, other):        # m1 + m2
        return Money(self.amount + other.amount, self.currency)

    def __eq__(self, other):         # m1 == m2
        return self.amount == other.amount and self.currency == other.currency

    def __lt__(self, other):         # m1 < m2
        return self.amount < other.amount

    def __len__(self):               # len(m)
        return int(self.amount)

    def __getitem__(self, key):      # m[key]
        return f"{key}: {self.amount}"

    def __iter__(self):
        yield self.amount; yield self.currency

    def __enter__(self):             # with Money(100):
        print("entering")
        return self
    def __exit__(self, *args):
        print("exiting")

# Most common dunders: __init__, __repr__, __str__, __eq__, __hash__,
# __add__, __sub__, __lt__/__le__/__gt__/__ge__, __len__, __getitem__,
# __iter__, __next__, __enter__/__exit__, __call__

Abstract Base Classes

from abc import ABC, abstractmethod

class Pipeline(ABC):
    @abstractmethod
    def extract(self): pass

    @abstractmethod
    def transform(self, data): pass

    @abstractmethod
    def load(self, data): pass

    def run(self):
        data = self.extract()
        transformed = self.transform(data)
        self.load(transformed)

class MyPipeline(Pipeline):
    def extract(self): return [1, 2, 3]
    def transform(self, data): return [x * 2 for x in data]
    def load(self, data): print(data)

MyPipeline().run()

7. Iterators, Generators, Decorators High Interview Weight

Iterators

# Iterable: has __iter__ (list, dict, str, file)
# Iterator: has __iter__ AND __next__

class CountDown:
    def __init__(self, start):
        self.n = start

    def __iter__(self):
        return self

    def __next__(self):
        if self.n <= 0:
            raise StopIteration
        self.n -= 1
        return self.n + 1

for x in CountDown(5):
    print(x)              # 5,4,3,2,1

# Built-in iter() / next()
it = iter([1, 2, 3])
next(it), next(it), next(it)   # 1, 2, 3
next(it)                       # StopIteration

Generators — Memory-Efficient CRITICAL for DE

# Function with yield = generator
def countdown(start):
    while start > 0:
        yield start
        start -= 1

for x in countdown(5):
    print(x)

# Generator expression (like list comp, with parens)
gen = (x ** 2 for x in range(1_000_000))     # no memory used yet
next(gen)                                     # produces one at a time

# Lazy file reading
def read_large_file(path):
    with open(path) as f:
        for line in f:
            yield line.strip()

for line in read_large_file("10gb.log"):     # never loads all 10GB
    process(line)

# yield from (delegate to another generator)
def chained():
    yield from range(5)
    yield from "abc"
list(chained())   # [0,1,2,3,4,'a','b','c']

# send/throw/close (advanced)
def coro():
    val = yield "ready"
    yield f"got {val}"

c = coro(); next(c); c.send("hello")
Why generators matter in DE: Process 100GB files line-by-line with constant memory. Stream API responses page-by-page. Build data pipelines that pull data on demand.

Decorators

# A decorator wraps a function with extra behavior

def timer(func):
    import time
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.time()-start:.3f}s")
        return result
    return wrapper

@timer
def slow_function():
    import time; time.sleep(1)

slow_function()   # prints timing

# Equivalent to: slow_function = timer(slow_function)

# Decorator with arguments
def retry(max_attempts=3, delay=1):
    def decorator(func):
        import time
        def wrapper(*args, **kwargs):
            for attempt in range(max_attempts):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    if attempt == max_attempts - 1:
                        raise
                    print(f"Attempt {attempt+1} failed: {e}")
                    time.sleep(delay * (2 ** attempt))   # exponential backoff
        return wrapper
    return decorator

@retry(max_attempts=5, delay=2)
def fetch_data():
    return requests.get(url).json()

# Preserve original function metadata
from functools import wraps

def my_decorator(func):
    @wraps(func)              # preserves __name__, __doc__
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

9. Concurrency & Parallelism

The GIL (Global Interpreter Lock)

Python (CPython) has a GIL: only one thread executes Python bytecode at a time. Implications:

  • Threading good for I/O-bound work (network, files, DB) — GIL released during I/O
  • Multiprocessing needed for CPU-bound work (heavy computation)
  • asyncio single-threaded concurrency for I/O — most efficient for many concurrent network calls

concurrent.futures (Recommended for most cases)

from concurrent.futures import ThreadPoolExecutor, ProcessPoolExecutor

# I/O bound — use threads
def fetch(url):
    return requests.get(url).json()

urls = ["url1", "url2", "url3"]

with ThreadPoolExecutor(max_workers=10) as executor:
    results = list(executor.map(fetch, urls))

# More control with submit / as_completed
from concurrent.futures import as_completed

with ThreadPoolExecutor(max_workers=10) as executor:
    futures = {executor.submit(fetch, url): url for url in urls}
    for future in as_completed(futures):
        url = futures[future]
        try:
            print(url, future.result())
        except Exception as e:
            print(f"{url} failed: {e}")

# CPU bound — use processes
def cpu_heavy(n):
    return sum(i*i for i in range(n))

with ProcessPoolExecutor(max_workers=4) as executor:
    results = list(executor.map(cpu_heavy, [10**7, 10**7, 10**7, 10**7]))

10. Data Engineering Patterns Production-Ready

Pattern 1: Retry with Exponential Backoff

import time
from functools import wraps

def retry(max_attempts=3, base_delay=1, exceptions=(Exception,)):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(max_attempts):
                try:
                    return func(*args, **kwargs)
                except exceptions as e:
                    if attempt == max_attempts - 1:
                        raise
                    delay = base_delay * (2 ** attempt)
                    print(f"Attempt {attempt+1} failed: {e}. Retrying in {delay}s")
                    time.sleep(delay)
        return wrapper
    return decorator

# Or use tenacity library (production-grade)
from tenacity import retry, stop_after_attempt, wait_exponential

@retry(stop=stop_after_attempt(5), wait=wait_exponential(min=2, max=60))
def fetch_api():
    return requests.get(url, timeout=30).json()

from pathlib import Path
import yaml

class Config:
    def __init__(self, env=None):
        env = env or os.getenv("ENV", "dev")
        with open(f"config/{env}.yaml") as f:
            self._data = yaml.safe_load(f)

        # Env var overrides config file
        env_key = key.upper().replace(".", "_")
        if env_val := os.getenv(env_key):
            return env_val
        # Nested keys: db.host
        val = self._data
        for k in key.split("."):
            val = val.get(k) if val else None
        return val or default

Pattern 4: API Client (Reusable)

import requests
from requests.adapters import HTTPAdapter
from urllib3.util.retry import Retry

class APIClient:
    def __init__(self, base_url, token, timeout=30):
        self.base_url = base_url.rstrip("/")
        self.timeout = timeout

        self.session = requests.Session()
        self.session.headers.update({"Authorization": f"Bearer {token}"})

        # Built-in retry
        retry = Retry(
            total=3, backoff_factor=2,
            status_forcelist=[429, 500, 502, 503, 504]
        )
        self.session.mount("https://", HTTPAdapter(max_retries=retry))

    def get(self, path, **kwargs):
        url = f"{self.base_url}/{path.lstrip('/')}"
        r = self.session.get(url, timeout=self.timeout, **kwargs)
        r.raise_for_status()
        return r.json()

    def paginate(self, path, page_size=100):
        """Yield all pages — generator!"""
        page = 1
        while True:
            data = self.get(path, params={"page": page, "size": page_size})
            if not data["items"]: break
            yield from data["items"]
            page += 1

client = APIClient("https://api.example.com", os.environ["API_TOKEN"])
for record in client.paginate("/users"):
    process(record)

def chunked_reader(path, chunk_size=10000):
    """Yield chunks of lines for batch processing."""
    chunk = []
    with open(path) as f:
        for line in f:
            chunk.append(line.strip())
            if len(chunk) >= chunk_size:
                yield chunk
                chunk = []
        if chunk:
            yield chunk

11. Interview Questions & Practice

Top Conceptual Q&A

Q1. Difference between list, tuple, set, dict?
List: ordered, mutable, allows duplicates, indexable. Tuple: ordered, immutable, allows duplicates. Set: unordered, mutable, unique items, no indexing. Dict: ordered (Python 3.7+), mutable, key-value pairs, keys must be hashable.
Q2. What is the difference between == and is?
== compares values (calls __eq__). is compares object identity (same memory address). Use is only for None: x is None.
Q3. Mutable vs immutable types — give examples and why it matters?
Immutable: int, float, str, tuple, frozenset. Mutable: list, dict, set. Matters because mutables can change in place (memory-efficient but can cause unexpected side effects when shared). Default mutable args are a common bug.
Q4. What is the GIL?
Global Interpreter Lock — only one thread executes Python bytecode at a time. Threading helps with I/O-bound work (GIL released during I/O). For CPU-bound parallelism use multiprocessing.
Q5. Generators vs Iterators?
All generators are iterators, not vice versa. Iterators implement __iter__ and __next__. Generators are a simpler way to create iterators using yield. Generators are lazy and memory-efficient.
Q6. List comprehension vs generator expression?
List comp [x for x in y] builds the entire list in memory. Generator exp (x for x in y) produces items one at a time — use for big data.
Q7. What's a decorator? Real-world example?
A function that wraps another function to add behavior without modifying it. Examples: @retry for resilience, @cache for memoization, @property for getters, Flask's @app.route for routes.
Q8. Shallow vs deep copy?
Shallow (copy.copy or list.copy) copies outer object but shares inner references. Deep (copy.deepcopy) recursively copies everything. For nested lists/dicts always deepcopy.
Q9. What is __init__ vs __new__?
__new__ creates the instance (returns it). __init__ initializes the already-created instance. You rarely override __new__ (only for metaclasses, singletons).
Q10. Difference between staticmethod, classmethod, and instance method?
Instance method: takes self, accesses instance. Classmethod: takes cls, accesses class (alternative constructors). Staticmethod: no self/cls, just a function in class namespace (utility).
Q11. How do you handle large files in Python?
Stream line-by-line with for line in open(path), use generators for chunking, and read in binary chunks for non-text. Avoid read() on large files (loads everything to memory).
Q12. How do you make an HTTP request with retry?
Use requests with an HTTPAdapter mounting a Retry config, or use tenacity library with @retry(stop=stop_after_attempt(5), wait=wait_exponential()).
Q13. What is virtualenv? Why use it?
Virtual environment isolates Python packages per project so they don't conflict. Create with python -m venv venv, activate, then pip install. Pin versions in requirements.txt. Modern alternative: poetry or uv.
Q14. How do you write tests in Python?
Use pytest (industry standard). Write functions starting with test_. Use assert, fixtures for setup, parametrize for multiple cases, unittest.mock for mocking.
Q15. Difference between PEP8 and Black?
PEP8 is Python's style guide (recommendations). Black is an auto-formatter that enforces a consistent style. Most teams use Black + isort + ruff for linting.
